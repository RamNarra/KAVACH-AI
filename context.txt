1. WHAT THIS PROJECT IS
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

Kavach AI is a GenAI-powered Android APK malware analysis platform.

Users upload an APK via the web UI → the backend decompiles it with APKTool
and JADX → extracts the most security-relevant Java source files and the
AndroidManifest.xml → sends them to Gemini for static analysis → returns a
structured JSON risk report → saves it to Firestore → deletes the APK from
Firebase Storage immediately.

Core pipeline (each request, in order):
  1. Frontend uploads APK to Firebase Storage  (apks/<uid>/<timestamp>_<name>.apk)
  2. Frontend sends POST /api/analyze to backend with {apk_url, uid, email}
  3. Backend downloads APK via httpx
  4. APKTool decompiles APK → reads AndroidManifest.xml → extracts package name
  5. JADX decompiles APK → selects top 12 security-scored Java files (max 70k chars)
  6. Builds a detailed prompt and calls Gemini via google-genai SDK
  7. Parses JSON response (risk_score, threat_level, investigation_report)
  8. Saves result to Firestore collection "apk_analysis_results"
  9. Deletes APK from Firebase Storage (immediate cleanup)
  10. Deletes local temp folder

A. Gemini / Vertex AI SDK — MIGRATED to google-genai (>=1.16.0)
   - The OLD sdk was: import vertexai + from vertexai.generative_models import GenerativeModel
   - That is DEPRECATED as of June 24 2025, removal June 24 2026.
   - The NEW sdk is: from google import genai + genai.Client(vertexai=True, ...)
   - In requirements.txt: google-genai>=1.16.0  (installed: 2.6.0)
   - google-cloud-aiplatform is NO LONGER in requirements.txt — it was replaced.
   - Client is initialized ONCE at startup as global `genai_client`.
   - generate_content call: genai_client.models.generate_content(model=..., contents=..., config=...)
   - Config type: genai_types.GenerateContentConfig (NOT GenerationConfig)
   - response.text works identically to before.

B. Vertex AI location MUST be "global"
   - gemini-3.5-flash is NOT available on us-central1 or any regional endpoint
     for this GCP project/quota combination.
   - Setting location to us-central1 causes: 404 Publisher Model not found
   - LOCATION env var defaults to "global" in main.py.
   - Cloud Run env var LOCATION=global must also be set on the Cloud Run service.

C. Model name is exactly "gemini-3.5-flash"
   - NOT gemini-1.5-flash, NOT gemini-pro, NOT gemini-2.0-flash
   - Fallback model (if primary fails): gemini-1.5-flash (same client, same location)

D. APK storage cleanup — TWO-LAYER strategy
   Primary: Immediate deletion at the end of every analyze request (success or fail).
            This is in the `finally:` block of the /api/analyze endpoint in main.py.
            delete_storage_object(apk_url) is always called.
   Backup:  ops/cleanup_orphan_apks.py — scan bucket prefix "apks/", delete blobs
            older than 10 minutes. Run manually or schedule with cron / Cloud Scheduler.
            Run: python ops/cleanup_orphan_apks.py [--dry-run]
   There is NO 1-day lifecycle deletion policy as the primary strategy.
   storage-lifecycle.json (1-day rule) exists only as a coarse last-resort safety net.

E. No mock mode
   - All mock / USE_MOCK_ANALYSIS code has been permanently removed.
   - The backend always runs the real pipeline (APKTool + JADX + Gemini + Firestore).

F. No Next.js API routes for /api/analyze
   - The frontend's next.config.mjs has output: 'export' (pure static site).
   - There is no /frontend/src/app/api/ directory.
   - In development: frontend on localhost:3000 calls backend on localhost:8080 directly.
   - In production: Firebase Hosting rewrites /api/** to Cloud Run service kavach-api.
   - NEXT_PUBLIC_API_BASE_URL controls which backend the frontend hits.

G. Firestore created_at serialization fix
   - firestore.SERVER_TIMESTAMP is a Sentinel that FastAPI cannot serialize to JSON.
   - The document saved to Firestore uses SERVER_TIMESTAMP (correct for DB).
   - The HTTP response dict replaces it with: datetime.datetime.utcnow().isoformat() + "Z"
   - This prevents TypeError during FastAPI JSON response encoding.

H. Storage URL parsing — supports both URL formats
   - Firebase Storage download URL: https://firebasestorage.googleapis.com/v0/b/<bucket>/o/<encoded_path>?alt=media&token=...
   - GCS public URL: https://storage.googleapis.com/<bucket>/<path>
   - delete_storage_object() in main.py handles both formats.
   - It strips query params, percent-decodes the path, and extracts blob name.

I. Java file selection strategy
   - JADX decompiles to sources/ directory inside the temp folder.
   - select_key_java_files() filters by package name first (to skip androidx/kotlin/etc.)
   - Scores each file by counting security-relevant keywords.
   - Selects top 12 files, total capped at 70,000 characters to avoid token overflow.
   - MainActiviy gets +15 bonus score.

OLD (deprecated — DO NOT restore):
  import vertexai
  from vertexai.generative_models import GenerativeModel, GenerationConfig
  vertexai.init(project=PROJECT_ID, location=LOCATION)
  model = GenerativeModel("gemini-3.5-flash")

NEW (current — DO NOT revert):
  from google import genai
  from google.genai import types as genai_types
  genai_client = genai.Client(vertexai=True, project=PROJECT_ID, location=LOCATION)
  gen_config = genai_types.GenerateContentConfig(response_mime_type="application/json", temperature=0.1)
  ai_response = genai_client.models.generate_content(model="gemini-3.5-flash", contents=prompt, config=gen_config)

Why this was necessary:
  - vertexai.generative_models deprecated June 24 2025, scheduled for removal June 24 2026.
  - Source: https://discuss.google.dev/t/vertexai-sdk-packages-deprecation/192695
  - The old code still ran (no runtime failure) but showed UserWarning on every request.
  - Migrating to google-genai is Google's official recommended migration path.
